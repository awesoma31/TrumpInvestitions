    ### **Статус:** утверждено
    **Исполнитель:** группа до 10 человек
    **Руководитель:** Ключев А.О.
    **Дата:** [текущая дата]
    **Версия:** 2.0 (соответствует стеку ИТМО)

    ## 1. Основание для разработки
    Проект выполняется в рамках дисциплины «Разработка мобильных приложений» (Университет ИТМО). Цель — создание масштабируемой экосистемы для имитации биржевой торговли, способной обслуживать 10 000 клиентов.

    ## 3. Стек технологий

    Категория | Технологии | Примечание
    ---------|-----------|-----------
    **Мобильная разработка (Android)** | Kotlin, Jetpack Compose | Нативное приложение
    **Кроссплатформенная мобильная разработка** | React Native | Для iOS/Android
    **Бэкенд (основной)** | Kotlin, Ktor, корутины, kotlinx.serialization | Микросервис для связи с приложениями и работы с БД
    **Бэкенд (высоконагруженный)** | Go | Микросервис котировок
    **Системное программирование** | C | Драйвер для Linux (имитация биржи)
    **Базы данных** | PostgreSQL (SQL), ClickHouse (аналитика/история) | ClickHouse — для быстрой выдачи больших объёмов
    **Кэш / Брокер сообщений** | Redis Server / KeyDB | Кэширование, pub/sub
    **Мониторинг и логи** | OpenTelemetry | Сбор телеметрии
    **Финальный отчёт** | PDF (ГОСТ 7.32-2017), оглавление, введение, заключение, список литературы, диаграммы | Оформление как дипломная работа

>     **Важно:** отклонение от утверждённого стека ведёт к снижению балла. Все компоненты должны быть реализованы с использованием указанных технологий.

    ## 4. Функциональные требования

    ### 4.1. Мобильные приложения
    - **Android (Kotlin + Jetpack Compose):**
      - Экран авторизации (JWT)
      - Список акций (котировки в реальном времени)
      - Детальный просмотр акции (график, стакан, история)
      - Выставление лимитных и рыночных заявок
      - Просмотр портфеля и истории сделок
    - **React Native:**
      - Аналогичный функционал
      - Адаптация под iOS и Android

    ### 4.2. Бэкенд
    - **Микросервис на Ktor (шлюз для мобильных приложений):**
      - REST / WebSocket
      - Аутентификация, валидация, маршрутизация
    - **Микросервис на Kotlin (работа с БД):**
      - Пользователи, счета, заявки, сделки
      - PostgreSQL (нормализованная схема)
      - ClickHouse — хранение котировок и сделок для аналитики
    - **Микросервис на Go:**
      - Получение котировок от C-драйвера
      - Публикация в Redis (pub/sub)
      - Отдача через gRPC или HTTP/2
    - **Драйвер на C:**
      - Работает под Linux
      - Генерирует случайные изменения цен (симуляция биржи)
      - Передаёт данные в Go-сервис через сокет/shared memory

    ### 4.3. Имитатор нагрузки
    - Отдельный микросервис (Kotlin / Python / Go — по выбору команды)
    - Без GUI
    - Эмулирует 10 000 одновременных клиентов
    - Выполняет случайные сценарии: вход, подписка на котировки, выставление заявок

    ## 8. Требования к отчётности

    1. **Исходный код** — в Git-репозитории, структурированный, с комментариями.
    2. **Документация в коде** — Dokka для Kotlin-модулей.
    3. **Внутренняя документация** — Obsidian Vault (Markdown).
    4. **Финальный отчёт** — PDF, оформленный по ГОСТ 7.32-2017:
       - Титульный лист
       - Оглавление
       - Введение (актуальность, цель, задачи)
       - Аналитический обзор
       - Архитектура и проектирование
       - Реализация (схемы, скриншоты)
       - Тестирование (имитатор 10к)
       - Заключение
       - Список литературы
       - Приложения (при необходимости)
